Extreme Programming: Advantages and Disadvantages

Advantages Disadvantages
➕ Simplicity is one more advantage of Extreme Programming projects. The developers who prefer to use this methodology create extremely simple code that can be improved at any moment. ➖ Some specialists say that Extreme Programming is focused on the code rather than on design. That may be a problem because good design is extremely important for software applications.
➕ The whole process in XP is visible and accountable. Developers commit what they will accomplish and show progress. ➖ One more disadvantage of XP is that this methodology does not measure code quality assurance. It may cause defects in the initial code.
➕ XP assists to create software faster thanks to the regular testing at the development stage. ➖ XP is not the best option if programmers are separated geographically.
➕ The main advantage of Extreme Programming is that this methodology allows software development companies to save costs and time required for project realization.
